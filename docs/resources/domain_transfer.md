# openprovider_domain_transfer (Resource)

Manages domain transfer to OpenProvider. Transfers a domain from another registrar using an authorization code.

## Important Notes

- **Transfer Process**: Domain transfers typically take 5-7 days to complete. The resource is created once the transfer is initiated (status: `REQ`), not when it completes (status: `ACT`).
- **Auth Code**: The authorization code (EPP code) must be obtained from your current registrar before initiating the transfer.
- **Security**: The auth code is marked as sensitive and should be stored securely (e.g., using variables with encrypted state backend).
- **Delete Behavior**: Destroying this resource removes it from Terraform state only; the domain remains at OpenProvider.
- **Import Note**: After importing a transferred domain, you must provide the auth_code in your configuration, as it cannot be retrieved from the API.

### Required

- `auth_code` (String, Sensitive) The EPP/Authorization code for the domain transfer (also known as transfer code or auth code). This is obtained from the current registrar.
- `domain` (String) The domain name (e.g., example.com).

### Optional

- `admin_handle` (String) The admin contact handle for the domain.
- `autorenew` (Boolean) Whether the domain should auto-renew.
- `billing_handle` (String) The billing contact handle for the domain.
- `import_contacts_from_registry` (Boolean) Import contact data from registry and create handles after transfer. When enabled, contact handle parameters can be omitted.
- `import_nameservers_from_registry` (Boolean) Import nameservers from registry after transfer. When enabled, nameserver parameters can be omitted.
- `is_private_whois_enabled` (Boolean) Enable WHOIS privacy protection for the domain.
- `ns_group` (String) The nameserver group to use for this domain.
- `tech_handle` (String) The tech contact handle for the domain.

### Read-Only

- `expiration_date` (String) The domain expiration date.
- `id` (String) The domain identifier (domain name).
- `status` (String) The current status of the domain. Common values: REQ (transfer requested), ACT (active/completed).



## Import

Import an existing transferred domain using the domain name.

```shell
#!/bin/bash

# Import an existing transferred domain
terraform import openprovider_domain_transfer.example example.com
```

## Transfer Workflow

1. Obtain authorization code from current registrar
2. Create Terraform configuration with `openprovider_domain_transfer` resource
3. Run `terraform apply` to initiate transfer (status will be `REQ`)
4. Wait 5-7 days for transfer to complete
5. Run `terraform refresh` to update status to `ACT`
6. (Optional) Transition to `openprovider_domain` resource for ongoing management

## Status Values

- `REQ` - Transfer requested/in progress
- `ACT` - Transfer completed, domain is active
